Docker란?

컨테이너 가상화 플랫폼

애플리케이션 실행에 필요한 모든 것을 하나의 패키지(컨테이너)로 묶는 기술

왜 필요한가?

제 컴퓨터에서는 되는데요...
문제
PHP 버전 다름
MySQL 설정 다름

→
Docker
컨테이너에 환경 포함
→
해결

핵심 개념

Image (이미지) - 컨테이너의 설계도, 읽기 전용

Container (컨테이너) - 이미지를 실행한 인스턴스, 실제로 동작하는 환경

Volume (볼륨) - 데이터 저장소, 컨테이너 삭제해도 데이터 유지

기본 명령어

# 컨테이너 목록
docker ps

# 컨테이너 실행
docker run [이미지명]

# 컨테이너 중지
docker stop [컨테이너명]

# 이미지 목록
docker images

Laravel Sail이란?

Laravel 공식 Docker 개발 환경

Docker 설정을 직접 하지 않아도 Laravel 개발에 필요한 환경을 제공

포함된 서비스

  • PHP 8.x
  • MySQL 8.0
  • Redis
  • Meilisearch
  • Composer
  • Node.js / npm
  • Mailpit
  • Selenium

필요한 것만 선택해서 사용 가능

서버 관리 명령어

# 서버 시작 (백그라운드)
sail up -d

# 서버 중지
sail down

# 상태 확인
sail ps

개발 명령어

# Laravel 명령어 실행
sail artisan [명령어]

# Composer 패키지 설치
sail composer require [패키지명]

# npm 명령어
sail npm run dev

Docker vs Sail 명령어 비교

Docker 명령어를 직접 쓰면 복잡하지만, Sail이 이를 간단하게 감싸줍니다.

# 순수 Docker 명령어
docker exec -it salon-crm-laravel.test-1 composer require filament/filament

# Sail 사용
sail composer require filament/filament

컨테이너 이름, exec -it 옵션 등을 매번 입력할 필요 없이 sail 한 단어로 해결!

MVC 패턴

View
(화면)
←
Controller
(중간자)
→

(데이터)

  • Model: 데이터 구조와 비즈니스 로직
  • View: 사용자에게 보여지는 화면
  • Controller: Model과 View를 연결

디렉토리 구조

laravel-project/
├── app/
│   ├── Models/          ← 데이터 모델
│   ├── Http/Controllers ← 컨트롤러
│   └── Filament/        ← Filament 리소스
├── database/
│   └── migrations/      ← DB 설계도
├── resources/
│   └── views/           ← 화면 템플릿
├── routes/
│   └── web.php          ← URL 라우팅
└── .env                 ← 환경 설정

Artisan CLI

Laravel의 명령줄 도구

# 모델 생성
sail artisan make:model Customer

# 마이그레이션 생성
sail artisan make:migration create_customers_table

# 마이그레이션 실행
sail artisan migrate

# 캐시 초기화
sail artisan cache:clear

Eloquent ORM

Object-Relational Mapping

SQL을 직접 작성하지 않고 PHP 객체로 데이터베이스를 다룸

// SQL
SELECT * FROM customers WHERE id = 1;

// Eloquent
Customer::find(1);

// SQL
SELECT * FROM customers WHERE gender = '여';

// Eloquent
Customer::where('gender', '여')->get();

Filament이란?

Laravel Admin Panel 패키지

Model만 정의하면 CRUD 화면을 자동으로 생성

무료 / 오픈소스 / 활발한 커뮤니티

CRUD 자동 생성

CRUD = Create, Read, Update, Delete

Customer Model
↓
Filament Resource
↓
목록
List
생성
Create
수정
Edit
삭제
Delete

한 줄 명령어로 전부 생성

주요 컴포넌트

  • Resource - 하나의 Model에 대한 Admin 화면 세트
  • Form - 데이터 입력/수정 폼
  • Table - 목록 화면 (검색, 필터, 정렬)
  • Widget - 대시보드 컴포넌트 (통계, 차트)

설치 명령어

# 패키지 설치
sail composer require filament/filament

# 초기 설정
sail artisan filament:install --panels

# 관리자 계정 생성
sail artisan make:filament-user

# Resource 생성
sail artisan make:filament-resource Customer --generate

엑셀 vs 우리 시스템

항목 엑셀 Laravel + Filament
데이터 연결 VLOOKUP 수식 Relationship
데이터 무결성 아무거나 입력 타입 검증
